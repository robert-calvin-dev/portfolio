Why an Interactive Sandbox Matters

Learning CSS and JavaScript becomes much more effective when users can visually and instantly see changes. A color sandbox allows real-time experimentation with styles without having to refresh the page or edit static files manually.

Phase 1: Structure the Sandbox

  • 🖼️ Create a simple HTML layout with a variety of clickable elements.
  • 🎯 Add radio buttons to toggle between changing Background and Text colors.
  • 🎨 Integrate a color picker input to allow users to choose any color dynamically.

Phase 2: Code the Interactivity

🛠️ JavaScript Setup

Key components involve event listeners on the entire document body, color mode toggles, and resetting styles when needed.

document.addEventListener('DOMContentLoaded', function() {
    const colorPicker = document.getElementById('colorPicker');
    const resetBtn = document.getElementById('resetColors');
    const toggleBtn = document.getElementById('toggleSandbox');
  
    let sandboxActive = false;
    let originalStyles = new Map();
  
    toggleBtn.addEventListener('click', () => {
      sandboxActive = !sandboxActive;
      toggleBtn.textContent = sandboxActive ? '🔴 Deactivate Sandbox' : '🟢 Activate Sandbox';
      colorPicker.disabled = !sandboxActive;
      resetBtn.disabled = !sandboxActive;
    });
  
    document.body.addEventListener('click', (e) => {
      if (!sandboxActive) return;
      const target = e.target;
      const mode = document.querySelector('input[name="colorMode"]:checked').value;
      const styleProp = mode === 'background' ? 'backgroundColor' : 'color';
      if (!originalStyles.has(target)) {
        originalStyles.set(target, target.style[styleProp]);
      }
      target.style[styleProp] = colorPicker.value;
    });
  
    resetBtn.addEventListener('click', () => {
      for (const [element, original] of originalStyles.entries()) {
        element.style.backgroundColor = '';
        element.style.color = '';
      }
      originalStyles.clear();
    });
  });
